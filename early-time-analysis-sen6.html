When assessing early time heating data, it is important to use the lowest temporal resolution possible. For this report, every .xml file was read for each test to achieve this. Because of the inherent noise associated with reading the data in at this resolution, Kennelâ€™s fit_convolve() function was used to smooth the temperature responses shown in Figure 1.

Moving Time Interval

Figure 2 shows the evolution of thermal conductivity with a moving time window to calculate slope on log-time temperature plots. Paired with a plot of cubed differences between adjacent points which allows us to determine the best time frame to automatically choose monitoring interval boundaries based on this data signature.

Instantaneous Slopes

Figure 3 is generated using instantaneous slopes from the fit_convolve() function.

Comparing Tests

Comparing the early data from multiple A-DTS snapshots taken over the course of three years allows us to look at changes in back fill integrity with time. It is possible we could see changes in the monitoring interval length and decreases in slopes where bentonite is present indicative of integrity changes in the seals.

Considerations

  1. Because we are reading every .xml file from each test, there are a variety of different temporal resolutions summarized in the following table. For early time data, temporal resolution is important as this gives us a basis for how accurate the start of heating is determined. This should not make a large difference but is worth considering.

  1. One test(2018_08) uses a power output of 10W/min instead of 15W/min. This results in drastically different slopes which are accounted for when calculating thermal conductivity. Therefor it makes sense to compare the thermal conductivities instead of the slopes as we have done previously.
